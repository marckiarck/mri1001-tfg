# Final degree proyect in Computer Ingenieering in Burgos University: Development of a Platformer 2D video game using Unity

## Abstract
The intention of this work will be to develop a 2D platformer video game with the Unity game engine and the C# programming languaje.

The main mechanics of the game will consist of manipulating physics and time to help or obstruct the player to pass the levels of the game.

## My contribution to the project

This project was my **TFG**, so it is entirely developed by me, except for the artistic section that was made using free fair use art. The idea of the TFG was to develop a **video game** of the **2D Platforms** genre from start to finish, doing the necessary research to know what product I wanted to develop, writing all the necessary documentation to understand what game I wanted to develop and, finally, develop the game as afinal product, from the main elements such as the mechanics and the levels to those that are "background relative" such as the game options and sounds (SFX).

As a starting point for the project, I used project template called **Platformer Microgame**. This template brought the minimum elements necessary to build small levels of platform games. I made an evaluation of what it offered and how its code was structured and I made the decision to use it as a starting point (although later the particularities of my game made it necessary to rework almost everything again).

The project began by developing a **GDD** (Game Document Desing) in which I reflected how the game I was going to develop I wanted it to look like. A GDD explains how each element of the game is expected to be, from the gameplay and mechanics to the appearance of the menus.

With the GDD developed I was able to implement the game architecture used by the mechanics to achieve a fun gaming experience.

The entire process was documented in the ***Memory of the TFG*** and in the ***Annexes*** that accompany it. These are located in the *docs/Latex* folder. These documents are in ***Spanish***

# Trabajo de Fin de Grado del Grado en Ingeniería Informática: Desarrollo de un videojuego del género Plataformas 2D en Unity

## Resumen
La intención de este trabajo va a ser el desarrollo de un videojuego de plataformas 2D con el motor de videojuegos Unity y el lenguaje de programación C#.

Las principales mecánicas del juego consistirán en influir sobre la física y el tiempo para ayudar o dificultar que el jugador supere los niveles del juego.

## Mi contribución al proyecto

Este proyecto fue mi **TFG**, así que esta desarrollado enteramente por mí, salvo el apartado artístico que fue hecho utilizando arte gratuito fair use. La idea del TFG era desarrollar un **videojuego** del genero **Plataformas 2D** de inicio a fin, haciendo la investigación necesaria para saber que producto quería desarrollar, escribiendo toda la documentación necesaria para entender que juego se deseaba desarrollar y, finalmente, desarrollar el juego en su totalidad, tanto los elementos principales como la mecanicas y los niveles hasta los que se encuentran en segundo plano como las opciones de juego y los sonidos (SFX).

Como punto de partida para el proyecto se uso una plantilla de proyecto llamado **Platformer Microgame**. Esta plantilla traía los elementos mínimos necesarios para montar pequeños niveles de juegos de plataformas. Hice una evaluación de que ofrecía esa plantilla y como estaba estrucurado su código y tomé la decisión de usarla como punto de partida (aunque luego las particularidades de mi juego hicieron necesario rehacerlo casi todo de nuevo).

El proyecto empezó desarrollando un **GDD** (Game Document Desing) en el que plasmé como quería que fuese el juego que iba a desarrollar. En el se explica como se espera que sea cada elemento del juego, desde la jugabilidad y mecánicas hasta el aspecto de los menús.

Con el GDD desarrollado pude implementar la arquitectura del juego que desemboca en las mecánicas y una experiencia de juego divertida.

Todo el proceeso fue documentado en la ***Memoria del TFG*** y en los ***Anexos*** que la acompañan. Estos se encuentran en la carpeta *docs/Latex*.
